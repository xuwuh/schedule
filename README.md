# Локальный сайт для расписания

Локальный сайт для создания учебного расписания.


Для взаимодействия с пользователем реализован сайт, на котором посредством ввода информации о группах, аудиториях и преподавателях генерируется рассписание, которое можно скачать. Для случаев, когда заранее есть все данные предусмотрен шаблон exel-таблицы, который можно скачать с сайта и при заполнении загрузить, чтобы получить рассписание, сгенерированное на основе информации из файла. Для тестирования работы можно создать синтетические данные с помощью файла schedule_data_fake.py и выгрузить полученную xlsx таблицу на сайт.

Принцип работы алгоритма:

1. **Генерация данных**

Для тестирования работы алгоритма предусмотрен файл `schedule_data_fake.py`.
Он автоматически создаёт Excel-файл `schedule_data.xlsx` с фейковыми данными
об аудиториях, преподавателях, предметах и группах.

В начале задаётся количество создаваемых объектов:
- аудиторий;
- преподавателей;
- предметов;
- групп.

Генерация начинается с аудиторий. Для каждой аудитории создаются:
- уникальный `id`;
- номер кабинета;
- случайная вместимость;
- дополнительные характеристики: наличие экрана или технического оборудования.

После этого создаются предметы. Для каждого названия предмета формируются две
записи:
- лекция;
- семинар.

Для предметов случайно выбираются количество часов в неделю, длительность
занятия и дополнительные условия к аудитории.

Затем генерируются преподаватели. Их ФИО создаются с помощью библиотеки
`Faker`, а каждому преподавателю назначается случайный набор предметов и
количество рабочих часов в неделю. После распределения выполняется
дополнительная проверка: если какой-то предмет не получил преподавателя, он
добавляется случайному преподавателю.

Аналогично создаются группы:
- задаётся номер группы;
- выбирается количество студентов;
- назначается набор предметов.

После основного распределения проверяется, что каждый предмет назначен хотя бы
одной группе.

В конце данные объединяются в общий словарь и записываются в Excel-книгу.
Для каждого раздела создаётся отдельный лист:
- `аудитории`;
- `преподаватели`;
- `предметы`;
- `группы`.

Полученный файл используется как тестовая база для последующей конвертации и
создания расписания.

2. **Создание рассписания(scheduler.py):**

Алгоритм работает в пределах заданного учебного дня:
- дни: понедельник - пятница;
- начало дня: `09:00`;
- конец дня: `18:00`;
- перерыв между занятиями: 15 минут;
- академический час: 45 минут.

При размещении занятия учитываются следующие ограничения:

- одна группа не может одновременно посещать два занятия.
- один преподаватель не может вести два занятия одновременно.
- одна аудитория не может использоваться двумя занятиями одновременно.
- преподаватель должен быть связан с предметом.
- недельная нагрузка преподавателя не должна превышать указанное значение.
- вместимость аудитории должна покрывать размер всех групп занятия.
- аудитория должна быть свободна в выбранный интервал.

Предусмотрено, что семинары и лекции разворачиваются по разному: семинар создаётся отдельно для каждой группы, а лекция одного предмета объединяет все группы, которым она нужна. 
Составление рассписания основывается на колличестве часов предмета, необходимых для групп. Для каждого занятия составляется задача по его организации, после чего они сортируются. Сначала планировщик пытается разместить более длинные занятия, а после занятия предметов с большей недельной нагрузкой.

Для каждой задачи алгоритм перебирает:
- день недели;
- возможное время начала;
- свободного преподавателя;
- свободную аудиторию.

Сначала проверяется доступность всех групп. Если хотя бы одна группа уже занята
в этом интервале, вариант сразу исключается.

Затем выбирается преподаватель:
- он должен вести нужный предмет;
- он должен быть свободен;
- после добавления занятия его недельная нагрузка не должна превысить колличесвто его рабочих часов в неделю.

После этого выбирается аудитория:
- она должна быть свободна;
- она должна вмещать всех студентов. для лекций при равном приоритете выбирается аудитория с большей вместимостью;
- она сортируется по пригодности к условиям предмета(для некоторых предметов требуются дополнительные условия, такие как наличие экрана, технического оборудования).

Если вариант не нарушает ограничений, то рассчитывается его числовая оценка:

score = окна групп * 10 + окна преподавателя * 5 + приоритет аудитории * 100 + близость ко времени начала дня * 0.01 + текущая нагрузка преподавателя

После перебора всех вариантов алгоритм сохраняет тот, который обладает наименьшим score, после чего алгоритм:

1. сохраняет занятость каждой группы;
2. сохраняет занятость преподавателя;
3. сохраняет занятость аудитории;
4. увеличивает использованную нагрузку преподавателя;
5. добавляет занятие в `lessons`.

Если не было найдено подходящих вариантов, занятие добавляется в `unplaced_lessons`.
Там сохраняются группа, предмет, тип, длительность и общая причина: не найден свободный преподаватель, кабинет или интервал времени с учётом
ограничений.

3. **Динамическое изменение расписания(dynamic_scheduler.py)**

Алгоритм динамического расписания работает не с пустым расписанием, а с уже
созданным списком занятий. Его задача - внести изменение в готовое расписание и
по возможности сохранить остальные занятия максимально близко к исходному
варианту.

Перед обработкой изменения алгоритм:
- создаёт копию готового расписания;
- добавляет каждому занятию внутренний `_id`, чтобы выбранные на сайте занятия
  можно было изменить точно;
- считывает исходные данные о предметах, преподавателях, аудиториях и группах;
- приводит названия предметов и типы занятий к единому виду для поиска нужных
  записей в исходных данных.

Алгоритм работает в тех же границах учебного дня, что и основной планировщик:
- дни: понедельник - пятница;
- начало дня: `09:00`;
- конец дня: `18:00`;
- шаг перебора времени: 15 минут.

При динамическом изменении учитываются конфликты готового расписания:
- одна группа не может иметь два занятия в пересекающееся время;
- один преподаватель не может вести несколько занятий одновременно;
- одна аудитория не может быть занята несколькими занятиями одновременно.

Дополнительно проверяется корректность выбранных ресурсов:
- новый преподаватель должен вести выбранный предмет;
- новая аудитория должна вмещать группы занятия;
- требования предмета к аудитории должны выполняться, например наличие экрана
  или технического оборудования.

В зависимости от действия пользователя алгоритм может:
- заменить преподавателя у выбранных занятий;
- убрать занятие полностью;
- перенести занятие на свободное время;
- заменить предмет, преподавателя или аудиторию занятия.

Если выбранное изменение не конфликтует с текущим временем занятия, оно
сохраняется сразу. Если после изменения слот становится занятым, алгоритм ищет
новый вариант размещения.

Для переноса занятия алгоритм перебирает:
- день недели;
- возможное время начала;
- свободный интервал с учётом группы, преподавателя и аудитории.

Каждый допустимый вариант переноса получает оценку. В ней штрафуется отличие от
исходного расписания:
- перенос на другой день получает большой штраф;
- перенос на другое время оценивается по удалённости от исходного начала
  занятия;
- при сравнении нескольких изменений также учитываются замена преподавателя,
  аудитории и предмета.

Поэтому при наличии нескольких свободных вариантов выбирается тот, который
меньше всего меняет уже составленное расписание.

При замене преподавателя алгоритм сначала ищет преподавателя, который:
- связан с нужным предметом;
- свободен в текущем интервале занятия.

Если подходящий преподаватель есть, занятие остаётся на прежнем месте. Если
преподаватель найден, но текущий слот занят, занятие переносится в лучший
свободный интервал.

При добавлении окна алгоритм проверяет занятия группы в выбранное время. Если
окно пересекается с несколькими занятиями, он пытается перенести этот набор
занятий так, чтобы суммарное изменение расписания было минимальным. Если
освободить слот не удалось, изменение отклоняется.

После применения действия занятия снова сортируются по дню недели, времени,
группе и предмету. Результатом работы становится новое готовое расписание,
которое можно проверить и экспортировать так же, как расписание после работы
основного планировщика.

4. **Проверка расписания(cheker.py)**

После создания расписания на сайте оно автоматически проверяется запуском `checker.py`.
Проверка выполняется для уже готового списка занятий и не изменяет расписание.

Для каждого занятия алгоритм:
- считывает день недели и диапазон времени;
- переводит время начала и конца в минуты;
- разделяет группы, если в одном занятии указано несколько групп;
- получает преподавателя и аудиторию занятия.

После этого занятия распределяются по трём видам занятости:
- занятость групп;
- занятость преподавателей;
- занятость аудиторий.

Для каждого ресурса алгоритм сравнивает занятия одного дня. Конфликт
фиксируется, если интервалы времени пересекаются:

`start1 < end2` и `start2 < end1`

Таким образом проверка находит ситуации, когда:
- у одной группы одновременно стоят два занятия;
- один преподаватель назначен на два занятия в одно время;
- одна аудитория занята несколькими занятиями одновременно.

Если конфликт найден, алгоритм формирует сообщение с указанием ресурса,
занятий, дня недели и времени пересечения. Если формат занятия или времени
некорректный, такая запись также добавляется в список ошибок.

Занятия из `unplaced_lessons`, которые генератор не смог поставить в
расписание, выводятся отдельно и не считаются конфликтами готового расписания.

5. **Конвекторы**

Для перевода данных между форматами используются два конвектора:
- `excel_to_json.py`;
- `json_to_calendar.py`.

Файл `excel_to_json.py` преобразует Excel-книгу в JSON. Сначала алгоритм
определяет тип загруженного файла:
- если найдены листы с аудиториями, преподавателями, предметами и группами,
  файл считается исходной базой данных для генерации расписания;
- если найден лист с готовыми занятиями, файл считается готовым расписанием.

При обработке Excel-файла конвектор:
- считывает названия листов и заголовки колонок;
- пропускает пустые строки;
- проверяет обязательные числовые поля;
- преобразует списки id и дополнительных условий;
- формирует JSON-структуру, которую могут использовать остальные алгоритмы
  проекта.

Файл `json_to_calendar.py` преобразует готовое расписание из JSON в календарь
формата `.ics`.

Для каждого занятия он:
- определяет календарную дату по дню недели;
- разбирает время начала и конца;
- создаёт отдельное событие календаря;
- добавляет в событие предмет, тип занятия, группу, преподавателя и аудиторию.

Если нужно повторять занятия несколько недель, в событие добавляется правило
еженедельного повторения. После обработки всех занятий создаётся календарный
файл, который можно импортировать в календарное приложение.

## Что умеет сайт

- загрузка Excel-файла по шаблону;
- проверка, что файл соответствует шаблону;
- ручной ввод аудиторий, преподавателей, предметов и групп без ручного ввода id;
- автоматическое создание расписания;
- просмотр расписания таблицей на сайте;
- проверка конфликтов;
- скачивание расписания в форматах ICS, Excel и PDF.

## Описание алгоритмов

Краткие описания также есть в коментариях основных модулей:
- `scheduler.py` — построение списка задач занятий, перебор допустимых слотов и выбор лучшего варианта расписания;
- `schedule_data_fake.py` — генерация демонстрационной Excel-базы с аудиториями, преподавателями, предметами и группами;
- `dynamic_scheduler.py` — точечные изменения уже созданного расписания и поиск свободного времени при переносах;
- `checker.py` — нормализация занятий и проверка пересечений групп, преподавателей и аудиторий;
- `excel_to_json.py` — конвертация Excel с исходными данными или готовым расписанием в JSON, а также создание Excel-шаблона;
- `json_to_calendar.py` — конвертация готового JSON-расписания в календарь ICS.
